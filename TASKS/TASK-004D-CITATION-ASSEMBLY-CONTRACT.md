# TASK-004D — CITATION ASSEMBLY CONTRACT

## STATUS

APPROVED FOR IMPLEMENTATION

## 1. OBJECTIVE

Establish the deterministic Citation Assembly Contract.

This task creates the foundation for constructing source-backed citations from legal objects and source records.

The purpose is:

```text
Legal Objects
+
Source References
+
Source Versions
+
Locations
=
Deterministic Citations
```

This task assembles citations.

It does NOT:

- generate answers
- rank authorities
- interpret law
- perform legal reasoning

## 2. CORE PRINCIPLE

Every legal conclusion generated by the platform must ultimately be traceable to authoritative sources.

The Citation Assembly Contract provides the mechanism to:

- identify source
- identify version
- identify location
- identify authority

in a deterministic and auditable manner.

## 3. CITATION PHILOSOPHY

The platform must never generate:

> Trust me.

The platform must generate:

> Here is the authority.  
> Here is where it appears.  
> Here is the version reviewed.

Every citation must be reproducible.

## 4. CREATE MODULE

Create:

```text
backend/app/services/citation/
```

Suggested structure:

```text
citation/
├── contract.py
├── assembler.py
├── formatter.py
├── models.py
```

## 5. CITATION CONTRACT MODEL

Create:

**CitationResult**

Fields:

- citation_id
- source_document_id
- source_version_id
- legal_object_id
- authority_type
- source_title
- official_reference
- publication_date
- effective_from
- effective_to
- location_reference
- citation_text
- assembled_at
- assembler_version

Use Pydantic.

## 6. AUTHORITY TYPE ENUM

Initial values:

- STATUTE
- REGULATION
- GUIDANCE
- PUBLIC_RULING
- PRIVATE_RULING
- CASE
- TRIBUNAL
- TREATY
- ACCOUNTING_STANDARD
- OTHER

Keep extensible.

## 7. CITATION ASSEMBLER

Create:

**CitationAssembler**

Primary method:

- `assemble()`

Input:

- legal_object

Output:

- CitationResult

## 8. MINIMUM REQUIRED CITATION ELEMENTS

Every assembled citation must contain:

**Source Title**

Example:

```text
Value Added Tax Act
```

**Source Version**

Example:

```text
Finance Act 2024 Amendment Version
```

**Authority Type**

Example:

```text
Statute
```

**Location Reference**

Examples:

- Section 15
- Article 8
- Regulation 5
- Paragraph 12

## 9. LOCATION REFERENCE CONTRACT

The citation system must support:

- Section
- Article
- Regulation
- Part
- Chapter
- Schedule
- Paragraph
- Clause
- Subsection

No interpretation.

Only identification.

## 10. CITATION TEXT ASSEMBLY

Example output:

```text
Value Added Tax Act,
Section 15,
Version effective 1 July 2024.
```

Or:

```text
Income Tax Act,
Section 63(2),
Finance Act 2025 Version.
```

Formatting can evolve later.

Determinism is the priority.

## 11. SOURCE TRACEABILITY RULE

Citation assembly must fail if:

- `source_version_id` missing

or

- `location_reference` missing

unless explicitly allowed by contract.

No orphan citations.

## 12. VERSION AWARENESS

Citation assembly must always reference:

- specific source version

Never:

- latest version

implicitly.

Critical governance rule.

## 13. HASH SUPPORT

Each CitationResult should generate:

**citation_hash**

based on:

- source_version_id
- legal_object_id
- location_reference

Purpose:

- reproducibility
- integrity
- auditability

## 14. FORMATTER LAYER

Create:

**CitationFormatter**

Purpose:

Convert structured citation into display text.

Must remain separate from assembly logic.

Assembly and presentation are different concerns.

## 15. STORAGE

Do NOT persist citations yet.

This task establishes:

- contract
- assembler
- formatter

only.

Persistence comes later.

## 16. TEST REQUIREMENTS

Create:

- `test_citation_assembler.py`
- `test_citation_formatter.py`

Verify:

- citation generation
- source traceability enforcement
- version awareness
- location references
- hash generation
- formatting consistency

## 17. OUT OF SCOPE

Do NOT add:

- answer generation
- citation ranking
- authority weighting
- legal reasoning
- AI
- search
- retrieval
- topic classification
- cross-regime analysis
- user interface

Strictly prohibited.

## 18. ACCEPTANCE CRITERIA

TASK-004D complete when:

- citation contract exists
- assembler operational
- formatter operational
- version awareness enforced
- source traceability enforced
- hash generation operational
- tests passing

## 19. FINAL PRINCIPLE

This task establishes:

```text
Authority → Citation
```

It does NOT establish:

```text
Authority → Meaning
```

Meaning and reasoning belong to later layers.

The citation layer exists to guarantee traceability, reproducibility, and trust.
